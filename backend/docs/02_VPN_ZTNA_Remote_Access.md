# VPN / ZTNA 远程访问指南

## 常见问题（快速定位）
- **无法登录 VPN**：多为 MFA 失效 / 账号锁定 / 客户端版本过旧
- **连上 VPN 但访问不了内网**：多为路由/分流策略、DNS、或权限组未下发
- **只有某些站点不可达**：多为 ZTNA 策略限制（按角色/设备合规）

## 1. 远程访问模式
公司通常提供两种模式（以实际为准）：
- **全隧道 VPN**：所有流量进公司网络（更稳，带宽占用高）
- **ZTNA（Zero Trust）**：按应用/域名授权（更安全，体验更好）

## 2. 登录失败排查（按顺序）
1. 检查时间同步（系统时间偏差会导致 MFA 失败）
2. 检查 MFA 设备是否可用（更换手机需走“重置 MFA”流程）
3. 更新客户端到推荐版本
4. 若报 “account locked / too many attempts”：
   - 进入自助解锁页面（如有）
   - 或提交工单：`IT-ACCESS`（附错误截图与时间）

## 3. “连上但访问不了内网”排查
1. DNS：确认内网域名是否解析为内网 IP（例如 `*.corp`）
2. 访问受限：某些系统仅允许公司设备/合规设备
3. 权限组：确认你是否在对应的 ZTNA/VPN 访问组

## 4. 支持信息（提交工单时请附）
- 设备系统版本（macOS/Windows）
- 客户端版本号
- 出错时间点（精确到分钟）
- 目标地址（域名/IP）
- 错误截图/日志（可脱敏）
