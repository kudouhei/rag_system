# 数据安全：访问控制与最小权限（RBAC / ABAC）

## 目标
- 任何数据访问都遵循 **最小权限原则（Least Privilege）**
- 高敏数据访问必须 **可审计、可追溯、可撤销**

## 1. 常见角色与权限边界（示例）
- **Employee（默认员工）**：仅访问 L0/L1 数据与公共系统
- **Engineer（研发）**：在项目范围内访问必要资源；生产权限按需申请
- **SRE/On-call**：生产访问按值班窗口授权；关键操作需要二次审批/双人复核
- **Data Analyst**：优先只读权限；导出需审批且记录用途
- **Security/Compliance**：审计读取权限，但仍遵循最小范围与留痕

## 2. 访问授权流程（建议）
1. 明确资源与数据级别（L0–L3）
2. 明确访问目的（why）与期限（when）
3. 审批链：直属经理 → 系统 Owner →（必要时）安全/合规
4. 自动化授权：通过 IAM/RBAC 组发放（避免手工）
5. 到期自动回收（default 7/14/30 天）

## 3. 生产权限（Prod Access）红线
- 禁止共享账号
- 禁止在个人设备长期保存生产数据导出
- 生产数据库导出必须：
  - 明确字段范围
  - 脱敏/匿名化（如可行）
  - 设置有效期与加密存储

## 4. 审计要求（最少字段）
- 操作人、时间、资源、动作、结果、来源 IP/设备（如允许）
- 关联工单/变更单号（ticket/change id）
