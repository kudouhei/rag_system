# 账号、SSO 与权限申请（IT Helpdesk）

## 适用范围
- 员工入职、外包/实习生开通账号
- SaaS 账号、Git/CI、云账号、数据库只读等权限申请
- 账号锁定、MFA 丢失、SSO 登录异常

## 1. 账号体系（总览）
- **主账号**：公司邮箱（即 SSO 身份）
- **SSO**：Okta / Azure AD（以公司实际为准）
- **关键系统**：
  - Jira / Confluence（工单与知识库）
  - GitHub/GitLab（代码）
  - CI/CD（Build/Deploy）
  - Cloud（AWS/Azure/GCP）与 Kubernetes

## 2. 入职账号开通（标准流程）
1. HR 在入职系统创建员工记录（含部门、直属经理、入职日期）
2. IT 自动创建邮箱 + SSO 身份（通常 30–120 分钟）
3. 直属经理在“权限申请表”选择角色模板：
   - **Default-Employee**：基础权限
   - **Engineer-Dev**：研发常用权限
   - **Data-Analyst**：BI/只读数仓
4. 安全部门自动下发 MFA（首登必须绑定）

**SLA**：
- 工作日 09:00–18:00：2 小时内完成
- 其它时间：次工作日优先处理

## 3. 权限申请（最少权限原则）
提交权限申请时必须包含：
- 申请人、部门、直属经理
- 目标系统（例如 GitHub/Jira/K8s）
- 权限级别（只读/写入/管理员）
- 业务理由（为什么需要）
- 期限（临时权限默认 7/14/30 天）

审批链（示例）：
1) 直属经理审批  2) 系统 Owner 审批  3) 安全/合规（如涉及敏感数据）

## 4. 密码重置 / MFA 丢失
- **密码重置**：优先走自助重置（SSO 入口）
- **MFA 丢失**：提交 `IT-ACCESS` 工单，需二次身份验证（工号/经理确认等）

## 5. 常见错误码（示例）
- `MFA_REQUIRED`：未绑定 MFA 或策略要求重新验证
- `ACCESS_DENIED`：无权限/未在访问组/审批未完成
