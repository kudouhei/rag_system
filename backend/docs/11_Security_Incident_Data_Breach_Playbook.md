# 安全事件与数据泄露响应（Data Breach Playbook）

## 1. 什么是数据泄露
任何未授权访问、披露、丢失、破坏个人数据/机密数据的事件都可能构成泄露。

## 2. 处理目标（优先级）
1) **止血**（封禁/吊销/隔离）  
2) **评估影响**（数据类型、数量、时间窗、是否外泄）  
3) **通知与合规**（按法规/合同）  
4) **复盘与改进**（行动项）

## 3. 立即行动清单（前 30 分钟）
- 锁定证据：日志/告警/访问记录（注意不要扩大数据暴露）
- 吊销疑似泄露的密钥/令牌
- 隔离受影响系统（只读/降级/临时下线）
- 建立事件频道与记录人

## 4. 通知与时限（示例）
- 监管/客户通知是否需要、以及时限：由法务/合规判断
- 内部升级：涉及 L3（PII/支付/密钥）必须立即升级安全与合规

## 5. 复盘输出（最少字段）
- 影响范围与根因
- 检测为何没提前触发
- 预防措施（技术 + 流程）
- 密钥轮换与权限收敛结果
