# 银行案例：合规优先的 RAG 知识助手（面试演示用）

## 一句话定位
为银行一线客服/网点/运营/合规提供 **“可引用、可审计、可控风险”** 的知识问答与流程指引助手，降低平均处理时长（AHT）并减少合规抽检问题。

## 典型问题（可直接拿来现场演示检索）
1. “客户称转账失败，返回 `AC04`/`AM09`，该怎么排查？需要收集哪些信息？”
2. “客户想提高转账限额，需要哪些身份核验材料？哪些情况必须升级合规？”
3. “信用卡争议（chargeback）流程是什么？每一步对应的时限要求？”

## 关键约束（银行‘红线’）
- **必须给引用**：所有结论必须指向政策/流程文档片段
- **不能编造**：检索不足时先澄清或拒答/升级
- **不落盘敏感信息**：日志/反馈仅保存脱敏后的内容

## 推荐的数据源（企业真实会有）
- 产品/费率说明（版本化）
- 操作手册（支付退回、争议、投诉、冻结/解冻流程）
- AML/KYC 政策（适用场景、材料清单、升级条件）
- 内部 QA/FAQ（经过合规审核）

## 为什么 RAG（而不是纯 LLM）
银行知识更新频繁且需溯源；RAG 可以：
- 让答案 **可解释（citations）**
- 将合规文档作为“事实来源”
- 结合角色与产品线做权限过滤
