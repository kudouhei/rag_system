# 数据安全：日志脱敏与留存（Logging, Redaction, Retention）

## 1. 日志里禁止出现的内容（示例）
- 明文密码、访问令牌、私钥、Session
- 完整银行卡号/IBAN、身份证件号
- 完整客户数据导出内容

## 2. 建议的日志策略
- **默认最小化**：只记录排障必需字段
- **分级日志**：
  - INFO：流程与关键指标
  - WARN/ERROR：错误码、trace_id、摘要（避免 full payload）
- **采样**：高 QPS 场景对请求体日志做采样

## 3. 脱敏方式（常见）
- 屏蔽：`a***@b.com`、`**** **** **** 1234`
- 哈希：对需要关联的标识（如 user_id）用不可逆哈希
- tokenization：对敏感字段映射到内部 token（需受控表）

## 4. 留存（Retention）建议（示例）
- 应用日志：7–30 天（按合规要求）
- 审计日志：180–365 天（或更久，按行业）
- 生产数据导出：尽量不留存；如必须留存要加密并设置到期销毁

## 5. 删除与合规请求（GDPR/DSAR）
当接到“删除/导出个人数据”请求时：
- 先确认身份与授权
- 记录处理时间线与证据
- 删除应覆盖：主数据、备份策略（按法规可有延迟）
