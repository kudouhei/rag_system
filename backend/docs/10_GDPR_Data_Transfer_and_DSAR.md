# 合规：GDPR、数据出境与 DSAR（数据主体权利）

> 说明：本文件用于内部知识库的“工作指引”，不是法律意见。实际以法务/合规部门要求为准。

## 1. 什么时候算“个人数据”（Personal Data）
只要能直接或间接识别自然人，通常都算个人数据：
- 邮箱、手机号、姓名、设备标识符
- IP 地址（在部分场景下也可能被视为个人数据）

## 2. 数据处理的合法性基础（示例）
- 合同必要性（提供服务必需）
- 合法利益（需评估平衡测试）
- 同意（可撤回，需可追溯）

## 3. 数据出境/跨境传输（跨区域共享）
常见触发：
- 把欧盟用户数据发到非 EU 区域处理
- 使用第三方 SaaS（服务器在其它区域）

建议检查清单：
- 数据类别与分级（L0–L3）
- 是否存在替代方案（本地化/匿名化）
- 法律机制：SCC / adequacy decision（按法务建议）
- 安全措施：加密、访问控制、审计、最小化

## 4. DSAR（数据主体访问/删除请求）处理流程（示例）
1. 接收请求 → 验证身份 → 建立工单（记录时间线）
2. 范围界定：哪些系统/哪些数据类型
3. 导出/删除执行：由系统 Owner 执行并留证
4. 回覆用户：在 SLA 内完成（按法规要求）

需要保留的证据（不含敏感数据明文）：
- 请求来源与身份验证记录
- 执行动作（导出/删除）与时间
- 涉及系统列表与负责人
