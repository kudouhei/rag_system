# 交叉编码器重排序技术

## 双阶段检索架构

现代高精度检索系统通常采用"粗排 + 精排"的两阶段架构：

1. **粗排阶段**：使用双编码器（Bi-Encoder）快速从全量文档中召回 Top-K 候选（通常 20-100 篇）
2. **精排阶段**：使用交叉编码器（Cross-Encoder）对候选文档精细打分，输出最终排序结果

## 双编码器 vs 交叉编码器

| 特性 | 双编码器 | 交叉编码器 |
|------|---------|-----------|
| 编码方式 | 查询和文档分别编码 | 查询和文档联合编码 |
| 速度 | 极快（可预计算文档向量） | 较慢（需实时计算） |
| 精度 | 中等 | 高 |
| 适用阶段 | 粗排召回 | 精排重排序 |

## 交叉编码器工作原理

交叉编码器将查询和候选文档拼接后输入 BERT 类模型，在 [CLS] token 位置输出相关性分数。由于查询和文档在注意力机制中充分交互，能够捕捉细粒度的语义匹配信号。

```
输入：[CLS] 查询文本 [SEP] 文档文本 [SEP]
输出：相关性分数（0-1）
```

## 常用模型

- `cross-encoder/ms-marco-MiniLM-L-6-v2`：英文，轻量高效
- `BAAI/bge-reranker-base`：中文，基础版本
- `BAAI/bge-reranker-large`：中文，高精度版本

## 性能提升

在企业知识库检索任务中，引入交叉编码器重排序平均可将 MRR@10 提升 8-15%，NDCG@5 提升 5-12%。
