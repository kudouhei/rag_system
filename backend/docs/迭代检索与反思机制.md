# 迭代检索与反思机制

## ReAct 框架

ReAct（Reasoning + Acting）是一种让语言模型在推理过程中交替生成思考步骤和执行动作的框架。在 RAG 系统中，ReAct 框架被用于实现自适应的迭代检索：

1. **思考（Reasoning）**：分析当前检索结果是否满足需求，判断失败原因
2. **行动（Acting）**：执行检索、重写查询或调整策略
3. **观察（Observation）**：评估新检索结果的质量

## 迭代检索流程

当初次检索的置信度低于阈值时，系统触发反思机制：

### 失败分析
- 关键词选择不当：查询词与文档词汇不匹配
- 语义偏差：查询意图与文档内容语义距离过大
- 查询过于宽泛：缺乏区分性特征，召回噪声过多

### 查询重写策略
- **同义词扩展**：替换为领域内更标准的术语
- **意图细化**：添加限定条件，明确查询意图
- **分解查询**：将复杂问题拆解为多个子查询

### 策略切换
自适应模式下，系统在不同迭代轮次自动切换检索策略：
- 第1轮：混合检索（向量 + BM25）
- 第2轮：纯向量检索（捕捉语义相似性）
- 第3轮：纯 BM25 检索（精确关键词匹配）

## 效果评估

在 Natural Questions 数据集上的实验表明：
- 基线召回率（单次检索）：61%
- 迭代检索后：提升至 76%（+15%）
- 多策略融合后：提升至 79.4%（+3.4%）
