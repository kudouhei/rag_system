# 数据分级与隐私（Data Classification & Privacy）

## 1. 数据分级（示例）
### L0 Public（公开）
可公开发布：官网文案、公开 API 文档等

### L1 Internal（内部）
仅公司内部使用：一般流程文档、非敏感统计

### L2 Confidential（机密）
客户合同、内部财务、未公开路线图、内部账号信息等  
访问需按角色授权，严禁外发。

### L3 Restricted（受限/敏感）
个人信息（PII）、支付/账户信息、密钥、访问令牌、生产数据库导出等  
访问需最严格控制，必须可审计，保存与传输均需加密。

## 2. PII 示例
- 姓名、手机号、邮箱、身份证件号
- 银行账号/卡号、IBAN
- 精确定位信息、设备标识符（按地区法规）

## 3. 使用规则（摘要）
- 只收集完成任务所需的最少数据
- 工单与日志中避免粘贴完整敏感数据（用脱敏/哈希）
- 共享数据前确认级别与接收方权限

## 4. 面向 AI 系统的注意事项
- 知识库文档中禁止存储密钥/令牌/生产数据样本
- 反馈与审计中仅保存脱敏后的内容
- 涉及 L2/L3 数据的问题，优先引导到受控流程（审批/升级）
