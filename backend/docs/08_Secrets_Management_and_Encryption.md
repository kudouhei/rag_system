# 数据安全：密钥管理与加密（Secrets & Encryption）

## 1. 密钥/令牌的基本规则
- **禁止**把 API Key、Token、私钥写入 git
- **禁止**在工单/IM 里明文发送密钥
- 使用公司指定的 Secrets 管理（Vault / AWS Secrets Manager / Azure Key Vault 等）

## 2. 密钥生命周期（建议）
1. 创建：最小权限、短有效期优先
2. 分发：仅通过 Secrets 管理器注入（CI/CD、运行时）
3. 轮换：定期轮换（30/60/90 天，按风险）
4. 吊销：员工离职/泄露事件立即吊销

## 3. 加密建议（通用）
- **传输加密**：HTTPS/TLS（内部服务也建议 mTLS）
- **存储加密**：磁盘/对象存储加密 + 数据库 TDE（如支持）
- **字段级加密**：对 L3（PII/支付）中的关键字段按需加密

## 4. 常见误区
- 把“脱敏日志”当成安全：脱敏只能降低风险，不能替代访问控制与加密
- 把密钥放在 `.env` 并提交到仓库：这是高频事故原因

## 5. 工单模板（密钥泄露）
提交安全事件工单时请包含：
- 泄露范围（哪种密钥、影响哪些系统）
- 泄露时间窗口
- 已采取措施（吊销/轮换/封禁）
- 后续行动项与负责人
