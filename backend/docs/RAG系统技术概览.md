# RAG 系统技术概览

## 什么是 RAG

检索增强生成（Retrieval-Augmented Generation，RAG）是一种将大语言模型与外部知识库结合的技术架构。传统语言模型仅依赖训练数据生成答案，而 RAG 系统能够在推理时实时检索相关文档，显著提升答案的准确性和时效性。

RAG 系统的核心流程包括：文档索引建立、用户查询解析、相关文档检索、上下文构建、答案生成五个阶段。

## 关键组件

### 文档切分（Chunking）

将长文档切分为适合检索的小块。常见策略包括固定长度切分、段落切分、语义切分。切片大小通常在 200-800 个 token 之间，需要在上下文完整性和检索精度之间取得平衡。

### 向量编码（Embedding）

使用预训练的编码器模型将文本转化为高维向量，捕捉语义信息。BAAI/bge 系列模型在中文检索任务上表现优异，支持将查询和文档映射到同一语义空间进行相似度计算。

### 近似最近邻检索（ANN）

对于大规模文档库，使用 FAISS、Hnswlib 等向量数据库实现高效的近似最近邻搜索，在毫秒级时间内从百万文档中找到最相关的候选集。

### BM25 关键词检索

经典的稀疏检索算法，基于词频-逆文档频率（TF-IDF）的改进版本。擅长处理精确关键词匹配场景，与向量检索形成互补。

## 混合检索策略

研究表明，将向量检索与 BM25 检索结合的混合策略，通常比单一方法提升 3-8% 的召回率。常用的融合方式包括 RRF（Reciprocal Rank Fusion）和线性加权融合。
