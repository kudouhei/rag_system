# 发布与回滚（Release & Rollback）

## 1. 发布前检查清单
- 变更范围与风险评估（是否涉及 DB schema、权限、计费、风控）
- 监控与告警：关键指标已覆盖（错误率、延迟、队列堆积）
- 回滚策略：一键回滚是否可用？是否需要数据回滚？
- 灰度策略：按区域/租户/百分比放量

## 2. 发布步骤（通用）
1. 创建 Release 记录（版本号、变更摘要、负责人）
2. 合并到发布分支，CI 通过
3. 部署到 staging 并做冒烟测试
4. 生产灰度：1% → 10% → 50% → 100%
5. 全量后 30–60 分钟观察期

## 3. 回滚策略
### 3.1 什么时候回滚
- 错误率/延迟显著上升且无法快速定位
- P0/P1 影响确认

### 3.2 回滚注意事项
- DB 迁移：向前兼容优先；不可逆迁移需额外审批
- 缓存：必要时清缓存/降级策略
- 通知：更新状态页与内部频道
